# Google Sheets Teaching Template — Specification

## Purpose

Google Sheets are used as **internal teaching whiteboards** for Survive Accounting.

Each approved teaching asset generates a Google Sheet template that allows instructors and VAs to organize the content into a structured teaching format.

Sheets are used for:

• tutoring sessions  

• instructional video recording  

• teaching demonstrations  

• structured walkthroughs of accounting problems

These sheets are **internal production tools**, not the primary student-facing delivery system.

---

# Sheet Generation

Once a teaching asset is approved in the Teaching Asset Library, the system automatically generates a Google Sheet.

The sheet contains a standardized structure that allows VAs to format the teaching material quickly.

The sheet template is designed to minimize manual work.

---

# Standard Sheet Tabs

Each sheet contains the following tabs.

### Metadata

Contains structured metadata about the teaching asset.

Examples:

• asset code  

• course code  

• chapter number  

• exercise reference  

• links to internal asset pages  

• links to LearnWorlds resources

This tab is primarily used by automation and internal tools.

---

### Hidden Data

Contains raw content exported from the Teaching Asset Library.

Examples:

• problem text  

• solution text  

• highlighted problem text  

• journal entry data  

• worked steps  

• formulas  

• concept notes

This data is copied into teaching layouts during sheet preparation.

---

### Master Whiteboard

The primary teaching workspace.

This sheet contains the formatted problem layout used during:

• tutoring sessions  

• instructional videos

Sections typically include:

• problem text  

• journal entries or calculations  

• worked steps  

• concept notes  

• answer summary

This layout is optimized for readability during screen recording.

---

### Journal Entries

A dedicated sheet for presenting journal entries clearly.

Multiple journal entry formats may be used.

Examples:

• completed entries  

• partially completed entries  

• teaching templates with missing elements

These formats help demonstrate accounting logic during instruction.

---

### Worked Steps

Contains step-by-step explanations showing how the problem is solved.

Steps should be written clearly so students can follow the reasoning process.

The goal is to demonstrate **how to think through the problem**.

---

### Important Formulas

Contains formulas relevant to the problem.

Examples:

• financial ratios  

• valuation formulas  

• accounting calculations

This sheet reinforces key concepts students must memorize.

---

### Concepts

Contains conceptual explanations related to the accounting topic.

Examples:

• definitions  

• conceptual frameworks  

• accounting rules

Concept notes help connect the problem to broader course concepts.

---

### Exam Traps

Highlights common mistakes students make.

Examples:

• sign errors  

• incorrect account classifications  

• misunderstanding accounting rules

This sheet helps students recognize and avoid typical exam pitfalls.

---

# VA Workflow

Sheet preparation is typically handled by the **Sheet Prep VA**.

Responsibilities include:

• copying data from the Hidden Data tab  

• formatting the Master Whiteboard  

• organizing journal entries  

• structuring worked steps  

• ensuring visual clarity

Automation and templates are designed to minimize manual formatting.

---

# Whiteboard Design Principles

Sheets should prioritize:

• readability during screen recording  

• clear step-by-step explanations  

• minimal visual clutter  

• logical flow of information

The Master Whiteboard layout should support both:

• live tutoring  

• recorded instructional content

---

# Automation Goals

The system should progressively automate more parts of sheet generation.

Potential improvements include:

• automatic content placement  

• structured journal entry generation  

• automatic formula insertion  

• standardized teaching layouts

The long-term goal is to reduce manual sheet preparation as much as possible.

---

# Relationship to Asset Library

Each Google Sheet corresponds to a single teaching asset stored in the Teaching Asset Library.

The sheet should always reference the asset code to maintain traceability.
